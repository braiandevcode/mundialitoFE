## DOCUMENTATION

### 1. OBJECTIVE
Create and structure the project's `README.md`. This document must be the definitive reference for any collaborator to understand the purpose, prerequisites, and run the application quickly.

### 2. EXECUTION RULES
- Use clear, concise, and direct language.
- Structure the content to include: purpose, technologies, exact installation steps, and local environment deployment.
- Include usage examples and, where applicable, placeholders for screenshots.
- Add links to relevant additional documentation.
- Ensure the inclusion of a License section.

### 3. RESTRICTIONS (STRICTLY AVOID)
- Exposing sensitive data, credentials, or environment keys.
- Omitting installation instructions or practical usage examples.
- Generating outdated, ambiguous, or incomplete documentation.
